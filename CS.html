When I first started with this website, the biggest problem I encountered was spacing the content out. I was pleased at first, but when I looked up my website on my phone, I realised how the margins weren't spaced properly.

As you can see above, the spacing comes from percentages. I am currently working on converting all the pixels back into percentages because percentages work easily with everything. Pixels are not the same on every screen.



left

right

As you can see on the top, the (left, middle, and right) was all done using pixels for the spacing.

Another issue I had with this related to the sizing was when you would resize the browser. There would be words and pictures overlapping on each other, whereas on the big websites such as Facebook, Google, etc it would automatically rescale itself when the browser got smaller or bigger. I discovered another sizing method which was vw and vh. It stands for viewpoint width and viewpoint height. This would allow for things to shrink or enlarge depending on the browser




As you can see from these two pictures, the first one is cleanly finished and the second one has overlapping pictures and words. I didnt realize it at first, but when I went on my phone to look at it, it was completely different. As I was coding it on my computer, I was basing it off a moniter screen using pixels. Pixels doesnt allow the flexibility to automatically resize when the broswer gets resized. Now that it has been resized, viewing it on a phone browser will look the same as on a computer.





I started this website during the summer of 2016 and the it has grown a lot ever since. I've made some major adjustments on the sizing and scaling of the webpage. The coolest feature is probably my homepage. The homepage is created with five images. The feature is called parallax scrolling. This shows an illusion where the background image moves by the camera slower than foreground images.
